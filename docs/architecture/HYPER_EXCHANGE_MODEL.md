# 前后端数据交换模型

本文档说明数据面（解析管线）与控制面（UI 视图）的跨线程数据交互机制，核心目标为规避主线程事件循环阻塞。

## 生产端批处理机制

数据不逐包跨线程，采用定量/定时的双条件批处理机制：

- **缓冲容器**：管线内部预分配容量为 5000 的 `QVector<ParsedPacket>` (`BATCH_RESERVE_SIZE`)。
- **触发投递条件**：
    - 缓冲区积压数据量 ≥ 5000 条。
    - 或两次投递的时间间隔 > 150ms。
- **零拷贝移交**：满足条件后，将容器封装为 `QSharedPointer<QVector<ParsedPacket>>`，通过 Qt 信号跨线程投递，仅传递指针拷贝。

## 消费端异步缓冲

控制面接收信号后，数据进入二级缓冲队列等待消费：

- **无界缓冲存入**：前端主窗体将新数据推入 `std::deque<ParsedPacket>`，队列硬限 100,000 条。
- **定时主动拉取**：配置 200ms 间隔的 `QTimer`，将数据处理从被动事件驱动转为主动定时拉取 (`processPendingPackets`)。

## 渲染时间片控制

为防止大批量数据插入导致 GUI 假死，消费环节实施时间片控制：

- **5ms 执行限时**：消费 `deque` 的 `while` 循环受 `QElapsedTimer` 监控，单次执行耗时超过 5ms 即强制中断，余量推迟至下一周期处理。
- **动态行折叠**：当开启“合并重复行”时，引擎比对新报文与当前末行报文的关键特征 (源/目的 IP、协议)。若特征一致，仅更新时间戳及 `repeatCount` 计数器，跳过昂贵的 `beginInsertRows` 视图插入操作。
