# 全局线程模型与调度

本文档定义系统的物理线程拓扑、隔离边界及同步机制。系统计算与 I/O 任务被严格拆分至不同线程池。

## 网卡捕获线程

- **实体与机制**：独立 `std::thread`，运行于阻塞模式。
- **职责**：调用 `pcap_next_ex` 读取网卡数据，向无锁内存池 (`ObjectPool`) 申请内存并进行全链路唯一一次 `memcpy`。
- **调度动作**：计算 IP 头部异或哈希，将报文内存块指针压入对应工作线程的 `SPSCQueue`。

## 协议解析线程组

- **实体与机制**：基于 `QThread`，工作于高优先级状态。
- **物理核绑定 (CPU Affinity)**：调用 `pthread_setaffinity_np` 将线程与特定 CPU 物理核绑定，确保 AC 自动机状态机及当前报文数据驻留 L1/L2 缓存。
- **职责**：无锁出队，执行 L2-L4 字段解剖、黑名单匹配拦截及 DPI 深度特征扫描。

## 异步持久化线程

- **实体与机制**：独立 `std::thread`，采用 `std::mutex` 与 `std::condition_variable` 实现休眠唤醒。
- **防内存耗尽边界**：内存中的待落盘告警队列 (`alertQueue`) 施加 20,000 条硬上限，超载时直接丢弃告警对象。
- **I/O 批处理**：线程唤醒后，单次提取最高 1,000 条告警，使用 `BEGIN IMMEDIATE` 独占事务及 `COMMIT` 聚合写入 SQLite WAL 数据库。

## 控制面主线程

- **实体与机制**：运行 `QApplication::exec()` 事件循环的系统主线程。
- **职责**：执行 Qt 视图树重绘及用户交互操作。
- **边界约束**：禁止执行网络/磁盘 I/O 或高计算复杂度任务，单帧渲染任务必须限制在 16ms 窗口内。
